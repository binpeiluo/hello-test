# Multica Number Merge - 已知问题

## 高优先级

### KI-001: 合成动画缺失

- **描述**：方块合成时没有平滑的移动和放大动画，直接跳到最终状态
- **影响**：用户体验生硬，缺乏反馈感
- **临时方案**：使用震动反馈弥补视觉反馈的不足
- **修复计划**：实现基于 `requestAnimationFrame` 的动画系统

### KI-002: 音效系统未实现

- **描述**：所有操作都没有音效反馈
- **影响**：游戏体验不完整，缺少沉浸感
- **临时方案**：依赖震动反馈
- **修复计划**：集成微信 `wx.createInnerAudioContext()` API

### KI-003: 道具系统仅实现撤回

- **描述**：规则表中定义了三种道具（撤回、清除、刷新），但只实现了撤回功能
- **影响**：后期关卡难度偏高，缺少辅助手段
- **临时方案**：增加关卡最大步数
- **修复计划**：实现清除和刷新道具

## 中优先级

### KI-004: 每日挑战功能缺失

- **描述**：缺少基于固定种子的每日挑战模式
- **影响**：社交分享功能受限
- **临时方案**：无
- **修复计划**：实现 seeded PRNG 和每日挑战入口

### KI-005: 关卡选择界面缺失

- **描述**：UX 流程中规划了关卡选择界面，但当前直接从第一关开始
- **影响**：玩家无法选择已解锁的关卡
- **临时方案**：通过存档的 `currentLevel` 自动进入当前关卡
- **修复计划**：实现关卡选择网格 UI

### KI-006: 暂停功能缺失

- **描述**：游戏中没有暂停按钮和暂停界面
- **影响**：玩家无法临时中断游戏
- **临时方案**：依赖微信 `onHide` 生命周期自动保存
- **修复计划**：添加暂停按钮和暂停菜单

### KI-007: 分享截图未生成

- **描述**：分享功能只能分享文字，无法生成游戏截图
- **影响**：分享内容吸引力不足
- **临时方案**：使用预设的分享图片
- **修复计划**：使用 Canvas `toDataURL()` 生成截图

### KI-008: InputHandler 事件监听器泄漏

- **描述**：`destroy()` 方法使用 `bind()` 创建新的函数引用，无法正确移除事件监听器
- **影响**：切换关卡时可能产生重复的事件监听
- **临时方案**：当前影响较小，因为切换关卡频率低
- **修复计划**：在构造时存储绑定后的函数引用

## 低优先级

### KI-009: 存档数据无版本迁移

- **描述**：存档数据结构如果在版本更新中发生变化，缺少迁移逻辑
- **影响**：更新后可能丢失玩家进度
- **临时方案**：当前版本无此问题
- **修复计划**：在 SaveData 中添加版本号字段，实现迁移函数

### KI-010: 数据埋点未实现

- **描述**：架构文档中规划了 `analytics.ts`，但尚未实现
- **影响**：无法追踪用户行为数据
- **临时方案**：无
- **修复计划**：实现基础埋点（关卡开始、完成、失败、分享）

### KI-011: 5x5 棋盘方块尺寸偏小

- **描述**：在小屏设备上，5x5 棋盘的方块尺寸可能偏小
- **影响**：触摸操作准确度下降
- **临时方案**：当前渲染器根据屏幕尺寸自动计算
- **修复计划**：优化响应式布局算法

### KI-012: 连击分数加成未实现

- **描述**：规则表中定义了连击分数加成（2连击1.5x、3连击2x等），但代码中未实现
- **影响**：连击只影响目标进度，不影响实际分数
- **临时方案**：无
- **修复计划**：在 MergeEngine 中添加连击倍率计算
