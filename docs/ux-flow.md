# Multica Number Merge - UX Flow

## 首局体验流程

```
启动游戏 → 首页 → 选择关卡 → 游戏内 → 胜利/失败 → 结算 → 分享/重玩
```

## 详细流程

### 1. 启动游戏

**状态：** 冷启动
**时间：** < 3 秒

**流程：**
1. 显示游戏 Logo（0.5 秒）
2. 加载资源（1-2 秒）
3. 进入首页

**异常处理：**
- 加载失败：显示重试按钮
- 网络异常：提示但可继续游戏

### 2. 首页

**状态：** 主菜单
**元素：**
- 游戏标题
- 开始游戏按钮
- 关卡选择按钮
- 设置按钮
- 每日挑战入口

**交互：**
- 点击"开始游戏"：进入当前关卡
- 点击"关卡选择"：显示关卡列表
- 点击"设置"：显示设置面板
- 点击"每日挑战"：进入每日挑战

### 3. 关卡选择

**状态：** 关卡列表
**元素：**
- 关卡网格（5x4 布局）
- 每个关卡显示：关卡号、目标、星级、锁定状态
- 返回按钮

**交互：**
- 点击已解锁关卡：开始该关卡
- 点击锁定关卡：提示需要完成前置关卡
- 滑动：浏览更多关卡

**视觉反馈：**
- 已完成关卡：显示星级（1-3 星）
- 当前关卡：高亮显示
- 锁定关卡：灰色显示

### 4. 游戏内

**状态：** 游戏进行中
**元素：**
- 游戏棋盘（居中）
- 分数显示（顶部）
- 关卡目标（顶部）
- 剩余步数（顶部）
- 道具栏（底部）
- 暂停按钮（右上角）

**交互：**
- 滑动：移动方块
- 点击道具：使用道具
- 点击暂停：暂停游戏

**视觉反馈：**
- 方块移动：平滑动画
- 合成：放大 + 颜色变化
- 消除：闪烁 + 消失
- 连击：特效 + 分数飘字
- 无效操作：轻微震动

### 5. 暂停

**状态：** 游戏暂停
**元素：**
- 半透明遮罩
- 暂停标题
- 继续游戏按钮
- 重新开始按钮
- 返回首页按钮

**交互：**
- 点击"继续"：恢复游戏
- 点击"重新开始"：重新开始当前关卡
- 点击"返回首页"：返回首页

### 6. 胜利

**状态：** 关卡完成
**元素：**
- 胜利标题
- 最终分数
- 星级评价（1-3 星）
- 下一关按钮
- 重新开始按钮
- 分享按钮

**星级标准：**
- 1 星：完成目标
- 2 星：剩余步数 > 20%
- 3 星：剩余步数 > 50%

**交互：**
- 点击"下一关"：进入下一关
- 点击"重新开始"：重新开始当前关卡
- 点击"分享"：分享成绩

### 7. 失败

**状态：** 游戏失败
**元素：**
- 失败标题
- 失败原因（步数耗尽/无法移动）
- 最终分数
- 重新开始按钮
- 使用道具复活按钮（可选）

**交互：**
- 点击"重新开始"：重新开始当前关卡
- 点击"使用道具"：使用道具复活（观看广告）

### 8. 结算

**状态：** 游戏结束
**元素：**
- 本次游戏统计
- 最高分对比
- 关卡进度更新
- 返回首页按钮

**交互：**
- 点击"返回首页"：返回首页

## 首局引导

### 第一关教学

**目标：** 合成数字 16

**引导步骤：**
1. **欢迎界面：** "欢迎来到 Multica Number Merge！"
2. **目标说明：** "本关目标：合成数字 16"
3. **操作引导：** "滑动屏幕移动方块"
4. **合成引导：** "相同数字会合成更大的数字"
5. **完成引导：** "继续合成，达成目标！"

**引导方式：**
- 手指动画指示滑动方向
- 高亮目标方块
- 简短文字说明
- 可跳过引导

## 响应式布局

### 屏幕适配

| 屏幕类型 | 棋盘大小 | 方块大小 | 间距 |
|---------|---------|---------|------|
| 小屏 (< 375px) | 300px | 60px | 8px |
| 中屏 (375-414px) | 340px | 70px | 10px |
| 大屏 (> 414px) | 380px | 80px | 12px |

### 横屏处理

- **默认：** 竖屏模式
- **横屏：** 提示用户旋转设备
- **强制：** 锁定竖屏方向

## 无障碍设计

### 视觉辅助

- **高对比度：** 数字和背景对比度 > 4.5:1
- **大字体：** 关键信息字体大小 > 16px
- **颜色辅助：** 不仅依赖颜色区分信息

### 操作辅助

- **大点击区域：** 按钮最小 44x44 px
- **明确反馈：** 所有操作都有视觉反馈
- **错误容忍：** 无效操作不惩罚玩家
